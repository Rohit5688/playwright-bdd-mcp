`SandboxEngine` is the local V8-based execution environment that powers the `execute_sandbox_code` MCP tool. It processes DOM trees, runs AST queries, and performs pattern searches entirely on your machine — transmitting only the filtered results to the LLM.

:::tip[Why This Matters]
Without the SandboxEngine, every analysis task would require sending raw file contents to the LLM. A typical project scan costs **~15,000 tokens** naively. With the SandboxEngine, the same analysis costs **~300 tokens** — a 98% reduction.
:::

---

## 🏗️ Architecture

The SandboxEngine creates a `vm2`-style isolated V8 context for every `execute_sandbox_code` call:

```
AI Prompt → MCP Handler → SandboxEngine (local V8 isolate)
                                    ↓
                          forge.api.* calls (file reads, AST, grep)
                                    ↓
                          Result filtered/summarized in sandbox
                                    ↓
                          Only the return value → LLM context
```

**What stays local**: Raw file contents, full AST trees, XML hierarchies, node_modules data.  
**What goes to LLM**: Only the `return` value of your script.

## 🔌 The `forge.api` Interface

All data access within the sandbox goes through the `forge.api` namespace:

### `forge.api.readFile(relativePath)`
Reads a file relative to `projectRoot`. Returns the full content as a string.
```javascript
const tsconfig = await forge.api.readFile('tsconfig.json');
const config = JSON.parse(tsconfig);
return config.compilerOptions.target;
```

### `forge.api.listFiles(directory, globPattern)`
Lists files matching a glob pattern.
```javascript
const featureFiles = await forge.api.listFiles('./features', '**/*.feature');
return { count: featureFiles.length, files: featureFiles };
```

### `forge.api.searchFiles(regexPattern, directory)`
Regex search across all files in a directory. Returns `{ file, line, content }[]`.
```javascript
const results = await forge.api.searchFiles('@smoke', './features');
return results.map(r => r.file).filter((v, i, a) => a.indexOf(v) === i);
```

### `forge.api.analyzeCodebase(projectRoot, options)`
Runs full AST analysis and returns structured page objects / step definitions.
```javascript
const { existingPageObjects } = await forge.api.analyzeCodebase('/my-project', { type: 'pageObjects' });
return existingPageObjects.map(po => po.className);
```

### `forge.api.getConfig()`
Reads and parses `mcp-config.json`.
```javascript
const config = await forge.api.getConfig();
return { baseUrl: config.baseUrl, environment: config.currentEnvironment };
```

## ⚠️ Sandbox Constraints

| Constraint | Value | Reason |
| :--- | :--- | :--- |
| **Execution timeout** | 10,000 ms (default) | Prevents runaway scripts from blocking the MCP server |
| **Network access** | Disabled | Local-only processing; no external HTTP calls |
| **File writes** | Disabled | Use `validate_and_write` for all file mutations |
| **`require()`** | Restricted | Only `forge.api.*` namespace is available |
| **Max return size** | ~50 KB | Keep return values focused and filtered |

If you hit the timeout, filter your data inside the sandbox instead of returning everything:
```javascript
// ❌ Times out on large projects
const pages = await forge.api.listFiles('.', '**/*.ts');
return pages;  // Could be 1000+ files

// ✅ Filter first, return only what's needed
const pages = await forge.api.listFiles('.', '**/pages/**/*.ts');
return pages.slice(0, 20);  // Return only first 20
```

## 🧪 Common Patterns

<Tabs>
  <TabItem label="Find all @tags">
    ```javascript
    const files = await forge.api.listFiles('./features', '**/*.feature');
    const tags = new Set();
    for (const f of files) {
      const content = await forge.api.readFile(f);
      const matches = content.match(/@\w+/g) || [];
      matches.forEach(t => tags.add(t));
    }
    return [...tags].sort();
    ```
  </TabItem>
  <TabItem label="Find duplicate steps">
    ```javascript
    const steps = await forge.api.listFiles('./step-definitions', '**/*.ts');
    const stepMap = {};
    for (const f of steps) {
      const content = await forge.api.readFile(f);
      const matches = content.match(/Given|When|Then\(['"`](.*?)['"`]/g) || [];
      matches.forEach(m => {
        stepMap[m] = (stepMap[m] || 0) + 1;
      });
    }
    return Object.entries(stepMap).filter(([_, v]) => v > 1);
    ```
  </TabItem>
  <TabItem label="Check config completeness">
    ```javascript
    const config = await forge.api.getConfig();
    const required = ['baseUrl', 'currentEnvironment', 'browsers'];
    const missing = required.filter(k => !config[k]);
    return { valid: missing.length === 0, missing };
    ```
  </TabItem>
