## 🔧 Setup & Configuration Tools

### `setup_project`
Bootstraps an empty directory into a fully configured Playwright-BDD project.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Absolute path to the new or empty project directory |

**Returns**: `{ filesCreated: string[], message: string }`

**Usage**:
```
"Initialize a new TestForge project at /Users/me/my-app and use chromium"
```

### `repair_project`
Regenerates missing baseline files without overwriting custom code. Safe to run at any time.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Absolute path to the project |

**Returns**: `{ filesRegenerated: string[], skipped: string[] }`

**Usage**:
```
"Repair my TestForge project — some baseline files are missing"
```

### `upgrade_project`
Upgrades npm packages and migrates `mcp-config.json` to the latest schema. Safe to re-run.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `preview` | `boolean` | ❌ | Show what would change without writing (default: `false`) |

**Returns**: `{ log: string[], warnings: string[], packagesUpdated: number }`

### `manage_config`
Read, write, preview, or scaffold `mcp-config.json`. Use `write` for partial updates — it deep-merges, not overwrites.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `operation` | `"read" \| "write" \| "scaffold" \| "preview" \| "inject_app"` | ✅ | Operation type |
| `config` | `object` | ❌ | Partial config to merge in (for `write`/`preview`) |

**Returns**: Full config object (on `read`) or confirmation (on `write`/`scaffold`).

**Usage**:
```
"Update mcp-config.json to set baseUrl to https://staging.myapp.com"
```

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |

| `action` | `"read" \| "write" \| "scaffold"` | ✅ | Operation |
| `entries` | `{ key: string, value: string }[]` | ❌ | Key-value pairs to upsert (for `write`) |

**Usage**:
```
"Add TEST_USER=admin@example.com and TEST_PASS=secret to the .env file"
```

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |

## 🔬 Analysis Tools

### `execute_sandbox_code` ⚡ Turbo Mode
Executes JavaScript in an isolated V8 sandbox. **Use this instead of`analyze_codebase` for all analysis tasks** — 98% fewer tokens.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `script` | `string` | ✅ | JavaScript code to execute. Use `return` to return a value. Use `await forge.api.*()` to call services. |
| `timeoutMs` | `number` | ❌ | Execution timeout in ms (default: `10000`) |

**Available `forge.api` methods**:

| Method | Description |
| :--- | :--- |
| `forge.api.readFile(path)` | Read file contents as string |
| `forge.api.listFiles(dir, pattern)` | List files matching a glob |
| `forge.api.analyzeCodebase(root, opts)` | Run AST analysis |
| `forge.api.searchFiles(pattern, dir)` | Regex search across files |
| `forge.api.getConfig()` | Read mcp-config.json |

**Returns**: `{ success: boolean, result: any, executionTime: number }`

**Example**:
```javascript
// Count all @smoke tagged scenarios without reading every file
const files = await forge.api.listFiles('./features', '**/*.feature');
let smokeCount = 0;
for (const f of files) {
  const content = await forge.api.readFile(f);
  smokeCount += (content.match(/@smoke/g) || []).length;
}
return { smokeCount };
```

⚠️ **Token-heavy.** Use `execute_sandbox_code` instead for analysis. Use this only for initial project onboarding in small projects (< 5 files).

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `customWrapperPackage` | `string` | ❌ | Package name for a custom base page |

**Returns**: `{ existingSteps[], existingPageObjects[], existingUtils[] }`

---

### `scan_structural_brain`
Scans the import graph to identify God Nodes (files with > 5 incoming imports). Use before any large refactor.

**Returns**: Table of risky files that require extra caution when editing.

---

### `get_system_state`
Returns a snapshot of the current session: DOM scan history, token cost, and turn counter. Use to check if an async job is running.

**Returns**: `{ domHistory: Summary[], tokenCost: number, turnCount: number }`

---

### `get_token_budget`
Returns a per-tool breakdown of estimated token usage for the current session.

**Returns**: Formatted report with totals per tool call.

## 🔍 Inspection Tools

### `inspect_page_dom`
Launches a headless browser, navigates to a URL, and returns the Accessibility Tree. **Use this before generating Page Objects.**

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `url` | `string` | ✅ | Full URL to inspect |
| `storageState` | `string` | ❌ | Path to a Playwright storageState JSON to bypass login |
| `loginMacro` | `object` | ❌ | Automated login sequence before visiting the target URL |
| `waitForSelector` | `string` | ❌ | CSS selector to wait for before parsing (for slow SPAs) |
| `includeIframes` | `boolean` | ❌ | Also scrape iframes (default: `false`) |
| `projectRoot` | `string` | ❌ | Project root for loading config timeouts |

**loginMacro shape**:
```typescript
{
  loginUrl: string;
  userSelector: string;
  usernameValue: string;
  passSelector: string;
  passwordValue: string;
  submitSelector: string;
}
```

**Returns**: `{ accessibilityTree: Node[], pageTitle: string, url: string }`

**Usage**:
```
"Inspect the DOM at https://app.mysite.com/dashboard — log in first with admin@test.com / password123"
```

| `storageState` | `string` | ❌ | Path to storageState JSON (cookies/auth) |

**Returns**: Session context object.

### `navigate_session`
Navigates the persistent session to a new URL.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `url` | `string` | ✅ | Target URL |

## ✨ Generation Tools

### `generate_gherkin_pom_test_suite`
**The primary test generation tool.** Loads context from existing Page Objects, step definitions, and utilities, then returns a generation prompt that produces a full Playwright-BDD test suite.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `testDescription` | `string` | ✅ | Plain-English test intent |
| `projectRoot` | `string` | ✅ | Project root |
| `baseUrl` | `string` | ❌ | Override base URL for this generation |
| `customWrapperPackage` | `string` | ❌ | Custom base page package |

**Returns**: Generation prompt with context → LLM generates `.feature`, `Page.ts`, `steps.ts` as JSON → pass to `validate_and_write`.

**Usage**:
```
"Generate a Playwright BDD test suite for the checkout flow at https://shop.example.com — test add to cart, proceed to checkout, and enter payment details"
```

## 💾 Validation & Writing Tools

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `files` | `{ path: string, content: string }[]` | ✅ | Array of files to write |
| `pageUrl` | `string` | ❌ | URL for DOM re-inspection during self-healing retries |
| `dryRun` | `boolean` | ❌ | Audit/validate without writing (default: `false`) |

**Returns**: `{ valid: boolean, filesWritten: string[], errors?: string[] }`

## 🚀 Execution Tools

### `run_playwright_test`
Executes the Playwright-BDD test suite. Runs **asynchronously** by default — returns a `jobId` immediately.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `tags` | `string` | ❌ | Cucumber tag expression, e.g., `"@smoke"` or `"@login and not @skip"` |
| `overrideCommand` | `string` | ❌ | Custom run command (overrides `executionCommand` in config) |
| `specificTestArgs` | `string` | ❌ | Additional args like a specific feature file path |

**Returns (async)**: `{ status: "started", jobId: "job_xxx" }` → poll with `check_test_status`.

**Async polling pattern**:
```
1. "Run the @smoke tests"           → returns jobId
2. "Check status of [jobId]"        → check_test_status (wait 30s)
3. "Summarize the test results"     → summarize_suite (after completion)
```

---

### `check_playwright_ready`
Pre-flight check: verifies Playwright, browsers, `baseUrl`, and `mcp-config.json`.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `baseUrl` | `string` | ❌ | Override URL to check reachability |
| `forceRefresh` | `boolean` | ❌ | Ignore cache and run fresh checks |

**Returns**: `{ ready: boolean, checks: { name, status, message }[] }`

---

### `check_environment`
Full environment check: Node.js, Playwright, browsers, SDK paths, device availability, and config validation.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `baseUrl` | `string` | ❌ | Optional URL to test reachability |

**Returns**: `{ summary, ready, failCount, warnCount, checks[] }`

## 🔍 Quality Tools

### `audit_locators`
Scans Page Objects and flags brittle selectors. Returns a health score and Markdown report.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `pagesRoot` | `string` | ❌ | Relative path to pages dir (default: `"pages"`) |

**Selector health scoring**:
- ✅ `getByRole`, `getByLabel`, `getByTestId` — stable
- ⚠️ CSS class (`.my-class`) — fragile
- ❌ XPath — brittle, avoid

---

| `projectRoot` | `string` | ✅ | Project root |

**Returns**: `{ coveragePercent, missing[], actionableSuggestions[] }`

## 🔧 Healing Tools

### `self_heal_test`
Analyzes a test failure to determine if it's a scripting issue (fixable) or an application issue (a real bug). Returns targeted fix instructions.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `errorDna` | `object` | ✅ | The structured error block from `run_playwright_test` output |
| `pageUrl` | `string` | ❌ | Page URL — if provided, re-inspects the DOM automatically |
| `projectRoot` | `string` | ❌ | Project root |

**errorDna shape**:
```typescript
{
  code: "Infrastructure" | "Logic" | "Transient";
  causalChain: string;
  originalError: string;
}
```

---

### `heal_and_verify_atomically`
Orchestrator: heals → verifies → learns in a single call. Use when a test fails with a bad selector.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `error` | `string` | ✅ | Test failure error message |
| `xml` | `string` | ✅ | Current DOM snapshot or page URL context |
| `candidateSelector` | `string` | ✅ | The proposed replacement selector |
| `oldSelector` | `string` | ❌ | Original failed selector (for better learning) |

**Returns**: `{ healedSelector, verified, learned, confidence }`

---

### `verify_selector`
Tests a CSS/XPath selector live in the persistent browser session without running a full test.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `selector` | `string` | ✅ | Selector to test |
| `oldSelector` | `string` | ❌ | Original failed selector (to record a heal) |
| `projectRoot` | `string` | ❌ | Project root (required for `autoTrain`) |
| `autoTrain` | `boolean` | ❌ | Auto-record the fix in `mcp-learning.json` (default: `false`) |

**Returns**: `{ exists, displayed, enabled, tagName, text }`
