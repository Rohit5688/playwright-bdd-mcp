`PageController` is TestForge's internal DOM interaction service. It wraps the Playwright `Page` object with:
- **Accessibility-first element extraction**
- **Indexed element addressing** (the LLM references elements by index, not raw selectors)
- **Stability scoring** (prefers ARIA roles over brittle XPaths)

:::note[Internal Service]
`PageController` is not a directly-callable MCP tool. It is invoked automatically by `inspect_page_dom` and the Playwright test runner. This page documents its behavior to help you write better Page Objects that align with how TestForge discovers elements.
:::

---

## 🔍 Element Discovery Pipeline

When `inspect_page_dom` is called, `PageController` runs this pipeline:

1. **Full page load**: Waits for `networkidle` or the specified `waitForSelector`.
2. **Iframe expansion**: If `includeIframes: true`, recursively injects into each iframe's context.
3. **Accessibility Tree capture**: Calls Playwright's `page.accessibility.snapshot()`.
4. **Noise filtering**: Removes hidden, aria-hidden, decorative, and structural-only nodes.
5. **Stability scoring**: Tags each element with its recommended locator strategy.
6. **Index assignment**: Every actionable element gets a stable index for the session.

---

## 📊 Locator Strategy Priority

`PageController` always recommends the most stable available locator, in this order:

| Priority | Strategy | Playwright Method | Why |
| :---: | :--- | :--- | :--- |
| 1st | ARIA role + accessible name | `getByRole('button', { name: 'Submit' })` | Survives HTML restructuring |
| 2nd | Label text | `getByLabel('Email address')` | Tied to `<label>` contract |
| 3rd | Test ID (`data-testid`) | `getByTestId('submit-btn')` | Dev-controlled, stable |
| 4th | Placeholder text | `getByPlaceholder('Enter email')` | Input-specific |
| 5th | Visible text | `getByText('Sign In')` | Last resort — breaks on i18n |
| ❌ | XPath / CSS class | `page.locator('.btn-primary')` | Brittle — not recommended |

When you run:
```
"Generate a Page Object for the login screen"
```
`PageController` will produce locators using this priority — you don't need to specify it.

## 🖼️ Handling Dynamic Pages

For Single Page Applications (SPAs) or pages with delayed loading:

```
// Use waitForSelector in your prompt:
"Inspect https://app.mysite.com/dashboard, wait for '.dashboard-loaded' before capturing"
```

Internally, `PageController` calls:
```typescript
await page.waitForSelector('.dashboard-loaded', { state: 'visible' });
const snapshot = await page.accessibility.snapshot();
```

## 🪟 Iframe Handling

By default, `PageController` does not enter iframes. Enable with `includeIframes: true`:

```
"Inspect https://checkout.myapp.com — include iframes (the payment form is inside one)"
```

The extracted Accessibility Tree will include iframe content merged inline, with prefixed locators to distinguish them:

```typescript
// Generated Page Object for iframe elements
readonly paymentIframe = this.page.frameLocator('[data-testid="payment-frame"]');
readonly cardNumber = this.paymentIframe.getByLabel('Card number');
```
