The `PageController` is the low-level execution engine that interacts directly with browsers. It handles DOM extraction, element interaction, and the `SimulatorMask` visual overlay.

> [!NOTE]
> PageController is designed to be **LLM-Agnostic**. It provides a structured interface that any reasoning engine can consume.

## 🌳 DOM Pipline

### `getSimplifiedHTML()`
Returns a pruned, token-efficient representation of the current page.
- **Pruning Logic**:
    - Removes all `<script>`, `<style>`, and `<svg>` contents.
    - Filters out elements with `aria-hidden="true"`.
    - Collapses deeply nested `<div>` wrappers that don't increase semantic meaning.
- **Returns**: A simplified HTML string ready for LLM context windows.

### `getPageInfo()`
Extracts metadata about the current viewport and interactive state.
- **Fields**: `url`, `title`, `vitals` (LCP, FID), `interactiveElementsCount`.

## 🖱️ Interaction API

TestForge uses **Indexed Interactions**. Instead of passing raw CSS/XPath, the AI passes an `index` from the previously extracted tree. This prevents "Stale Element Reference" errors.

| Method | Parameters | Description |
| :--- | :--- | :--- |
| `clickElement` | `index: number` | Simulates a trusted human click on the element at the specified tree index. |
| `inputText` | `index, text` | Clears and types text into the specified input element. |
| `selectOption` | `index, value` | Selects a value from a dropdown or radio group. |
| `scroll` | `{ direction: 'up'\|'down' }` | Performs a natural scroll to bring elements into viewport. |

## 🎭 SimulatorMask (Visual Feedback)

When `enableMask: true` is set in the constructor, TestForge injects a semi-transparent overlay into the browser.

- **Blocking**: Prevents the human developer from accidentally clicking or interacting with the browser while the automation is running.
- **Visual Cues**: Highlights the element currently being targeted by the AI in **Cyan (00FBFF)**.
- **Auto-Dismiss**: The mask automatically fades out when the session is closed or a failure occurs.

## 🧠 Relevance Scoring Algorithm

`PageController` uses a proprietary scoring algorithm to decide which elements to surface to the AI.

```ts
score = (typeWeight * visibilityFactor) + (arability * depthPenalty)
```

1. **Type Weight**: Buttons, Inputs, and Links are given a baseline of 10.0.
2. **Visibility**: Elements in the current viewport receive a 1.5x multiplier.
3. **ARIA Priority**: Elements with `role` or `aria-label` receive a 2.0x boost.
4. **Depth Penalty**: Each level of nesting beyond 10 levels reduces the score by 0.5.
