# TASK-49 — AST Stub Hunter (Implementation Gate)

**Status**: TODO  
**Priority**: High (Integrity)  
**Complexity**: Medium  

## Problem
AI models sometimes generate "lazy code" containing comments like `// TODO: Implement this` or stub methods with empty bodies. If these are written to the codebase, they cause silent failures or runtime errors.

## Scope
- [ ] Create `ASTScrutinizer` utility in `src/utils/`.
- [ ] Implement a parser (e.g., using `ts-morph` or simple regex/line scans) to detect:
    - `// TODO` or `// FIXME`
    - Empty method bodies in Page Objects.
    - "Generated by AI" placeholders that aren't replaced.
- [ ] Integrate into `FileWriterService.validateAndWrite`.
- [ ] Hard-reject any batch containing a stub. Return a clear "Self-Healing Hint" to the AI.

## AppForge Benchmark
- `FileWriterService.ts` lines 94-111.
- `ASTScrutinizer.scrutinize(content, path)` is called before the `tsc` check.
- Result: 90% reduction in incomplete code merging.
